HTTP Routing

Basic Routing

The most basic Laravel routes simply accept a URI and a Closure, providing a very simple and expressive method of defining routes.

我们搜索网站的时候需要更详细的资源地址。Route提供了这些地址。

Route Parameters

Required Parameters

我们输入的地址中可能有可以利用的字段，将此字段作为参数。

Optional Parameters

参数值有默认，你可以选择输入新的参数值或者不输入。

Regular Expression Constraints

可以对输入参数进行限制

Global Constraints

为每一个路由的参数都设置正则限制有点冗余，可以设置全局的正则限制。

If you would like a route parameter to always be constrained by a given regular expression, you may use the pattern method. You should define these patterns in the boot method of your RouteServiceProvider

Named Routes

没看懂

Named routes allow the convenient generation of URLs or redirects for specific routes. You may specify a name for a route using the as array key when defining the route.

将会返回完整的链接。
好处在于：路径变化了，return的结果也跟着变。

Route Groups

Route groups allow you to share route attributes, such as middleware or namespaces, across a large number of routes without needing to define those attributes on each individual route.

为路由的共有属性设置，而不用每次重复设置。

HTTP Middleware

HTTP middleware provide a convenient mechanism for filtering HTTP requests entering your application.

这个定义不是很准确。下面的这个更好。

中间件是在网站和浏览器之间的层。可对请求和响应做出过滤处理。

Whether a middleware runs before or after a request depends on the middleware itself.

中间件可以在请求之前做出反应也可以在响应之后做出反应。

中间件发挥作用必须加入路由之中。
第一步：为中间件取一个名字
第二步：为路由赋予中间件

Instead of defining all of your request handling logic in a single routes.php file, you may wish to organize this behavior using Controller classes. Controllers can group related HTTP request handling logic into a class. Controllers are stored in the app/Http/Controllers directory.

在路由中有一个函数用来处理请求，这个处理函数可以写在控制器当中。
所有的处理函数都写在控制器类中。
